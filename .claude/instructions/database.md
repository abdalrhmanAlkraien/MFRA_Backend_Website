## Core Rules — Never Violate

```
1.  Flyway only — spring.jpa.hibernate.ddl-auto = validate — never create/update
2.  Every table has deleted_at — soft delete is mandatory — never hard DELETE SQL
3.  UUID primary keys — never sequential integer auto-increment
4.  TIMESTAMPTZ for all timestamps — never TIMESTAMP (loses timezone info)
5.  NUMERIC(19,4) for money and decimals — never FLOAT or DOUBLE PRECISION
6.  NOT NULL on every required field — never rely on application checks alone
7.  Index every column used in WHERE, JOIN, or ORDER BY
8.  Every query must filter deleted_at IS NULL — never return soft-deleted records
9.  Instant (UTC) in Java for timestamps — never LocalDateTime
10. Foreign keys declared explicitly in both migration SQL and Java entity
```

## Primary Key Convention

```sql
-- Always UUID, always generated by the DB
id UUID PRIMARY KEY DEFAULT gen_random_uuid()
```

```java
// Always in Java entity
@Id
@GeneratedValue(strategy = GenerationType.UUID)
private UUID id;
```

**Why UUID:** Globally unique, safe to expose in URLs, no sequence contention
in distributed environments, not guessable.

**Never use:**
```sql
id SERIAL PRIMARY KEY         ❌ sequential, guessable
id BIGSERIAL PRIMARY KEY      ❌ sequential, guessable
id INT GENERATED ALWAYS AS IDENTITY  ❌ sequential
```

## Timestamp Convention

**In SQL:**
```sql
created_at  TIMESTAMPTZ NOT NULL DEFAULT now()
updated_at  TIMESTAMPTZ NOT NULL DEFAULT now()
deleted_at  TIMESTAMPTZ                        -- NULL = active
published_at TIMESTAMPTZ                       -- NULL = not published
```

**In Java:**
```java
private Instant createdAt;     // ✅ Instant — always UTC
private Instant updatedAt;     // ✅ Instant — always UTC
private Instant deletedAt;     // ✅ Instant — null = active
private Instant publishedAt;   // ✅ Instant — null = not published

// ❌ Never use
private LocalDateTime createdAt;   // loses timezone
private LocalDate createdAt;       // loses time
private Date createdAt;            // legacy, avoid
```

## Audit Columns — Required on Every Table

```sql
created_at  TIMESTAMPTZ NOT NULL DEFAULT now(),
updated_at  TIMESTAMPTZ NOT NULL DEFAULT now(),
created_by  UUID,                               -- who created it
updated_by  UUID,                               -- who last modified it
deleted_at  TIMESTAMPTZ                         -- soft delete timestamp
```

All five columns on every table without exception.

## BaseEntity — Extend on Every Entity

```java
@MappedSuperclass
@EntityListeners(AuditingEntityListener.class)
public abstract class BaseEntity {

    @Id
    @GeneratedValue(strategy = GenerationType.UUID)
    private UUID id;

    @CreatedDate
    @Column(nullable = false, updatable = false)
    private Instant createdAt;

    @LastModifiedDate
    @Column(nullable = false)
    private Instant updatedAt;

    @CreatedBy
    private UUID createdBy;

    @LastModifiedBy
    private UUID updatedBy;

    private Instant deletedAt;    // null = active, non-null = soft deleted
}
```

**Every entity class must extend BaseEntity — no exceptions.**
Never redeclare these fields in a subclass.

## Column Type Mapping

| Data | SQL Type | Java Type | Notes |
|---|---|---|---|
| Short label / name | `VARCHAR(100)` | `String` | Max 100 chars |
| Title / heading | `VARCHAR(255)` | `String` | Max 255 chars |
| URL / path | `VARCHAR(500)` | `String` | Long enough for S3 URLs |
| Long content / body | `TEXT` | `String` | No length limit |
| Status / category | `VARCHAR(50)` | `String` / enum | See Enum Pattern |
| Integer counter | `INT` | `Integer` | Order, count, position |
| Money / decimal | `NUMERIC(19,4)` | `BigDecimal` | **Never FLOAT** |
| Boolean flag | `BOOLEAN` | `Boolean` | |
| Timestamp | `TIMESTAMPTZ` | `Instant` | **Never TIMESTAMP** |
| Date only | `DATE` | `LocalDate` | Only when time irrelevant |
| UUID reference | `UUID` | `UUID` | Always + FK constraint |
| Flexible JSON | `JSONB` | `String` | Only truly dynamic data |
| Simple array | `TEXT[]` | `List<String>` | Use sparingly |

## NOT NULL Convention

```sql
-- Required field the user must provide:
title       VARCHAR(255) NOT NULL

-- Optional field the user may leave empty:
summary     TEXT                    -- nullable, no NOT NULL

-- Auto-generated field:
created_at  TIMESTAMPTZ NOT NULL DEFAULT now()
slug        VARCHAR(255) NOT NULL UNIQUE   -- generated by app before insert

-- Status with a safe default:
status      VARCHAR(50) NOT NULL DEFAULT 'DRAFT'

-- Foreign key that is optional:
category_id UUID REFERENCES categories(id)    -- nullable = belongs to no category

-- Foreign key that is required:
user_id     UUID NOT NULL REFERENCES users(id) -- always required
```

**Rule:** If the field appears in the design as required → `NOT NULL`.
If the field is optional in the UI → nullable (no NOT NULL).

## Flyway Migration Rules

### File Naming

```
V1__init_schema.sql
V2__create_auth_tables.sql
V3__create_<module>_tables.sql        ← one migration per module
V4__add_<column>_to_<table>.sql       ← additive change
V5__create_<junction_table>.sql       ← separate if large
```

**Rules:**
```
✅ Version number is sequential — never reuse or skip
✅ Name is descriptive — create_blog_tables not init_v2
✅ One file per logical change — do not bundle unrelated tables
✅ Never modify a migration that has already run in any environment
✅ New change = new migration file always
❌ Never delete a migration file
❌ Never rename a migration file after it has run
```

### Migration File Template

```sql
-- VX__create_<module>_tables.sql
-- Extracted from: designs/[N]-requirements.md
-- Module: [module name]

-- ─── Table: <table_name> ─────────────────────────────────────────

CREATE TABLE <table_name> (
    -- Primary key — always first
    id           UUID         PRIMARY KEY DEFAULT gen_random_uuid(),

    -- Domain fields (from extracted requirements)
    <field>      <TYPE>       [NOT NULL] [DEFAULT <value>],
    <field>      <TYPE>       [NOT NULL],

    -- Status column (if the design shows status badges)
    status       VARCHAR(50)  NOT NULL DEFAULT '<default_value>',

    -- Foreign keys (if relationships were identified)
    <parent>_id  UUID         [NOT NULL] REFERENCES <parent_table>(id),

    -- Audit columns — always last, always present
    created_at   TIMESTAMPTZ  NOT NULL DEFAULT now(),
    updated_at   TIMESTAMPTZ  NOT NULL DEFAULT now(),
    created_by   UUID,
    updated_by   UUID,
    deleted_at   TIMESTAMPTZ
);

-- Indexes — one per filterable/searchable/ordered column
CREATE INDEX idx_<table>_<column>  ON <table_name>(<column>);
CREATE INDEX idx_<table>_active    ON <table_name>(deleted_at)
    WHERE deleted_at IS NULL;
```

## Index Decision Rules

**Always add an index when:**

```
Column used in WHERE clause            → CREATE INDEX idx_<t>_<col> ON <t>(<col>)
Column used in ORDER BY                → CREATE INDEX idx_<t>_<col> ON <t>(<col>)
Column used in JOIN                    → CREATE INDEX idx_<t>_<col> ON <t>(<col>)
Foreign key column                     → Always index FK columns
Slug column (public URL lookup)        → CREATE UNIQUE INDEX
Status column (filtered in admin UI)   → CREATE INDEX idx_<t>_status ON <t>(status)
Active records filter                  → CREATE INDEX idx_<t>_active ON <t>(deleted_at)
                                           WHERE deleted_at IS NULL
Full-text search                       → CREATE INDEX ... USING GIN(to_tsvector(...))
```

**Skip the index when:**
```
Column is only SELECTed, never filtered
Column has very low cardinality (boolean true/false)
Table has fewer than ~1000 rows
```

**Mandatory indexes on every table:**
```sql
-- Active records partial index — always
CREATE INDEX idx_<table>_active ON <table>(deleted_at)
    WHERE deleted_at IS NULL;

-- Status index — whenever table has a status column
CREATE INDEX idx_<table>_status ON <table>(status);

-- Slug index — whenever table has a slug column
CREATE UNIQUE INDEX idx_<table>_slug ON <table>(slug);

-- FK indexes — on every foreign key column
CREATE INDEX idx_<table>_<parent>_id ON <table>(<parent>_id);
```

## Soft Delete Pattern

**Always soft delete — never hard delete.**

```java
// ✅ Correct — soft delete in service
entity.setDeletedAt(Instant.now());
repository.save(entity);

// ❌ Wrong — hard delete — never do this
repository.delete(entity);
repository.deleteById(id);
```

```java
// ✅ Correct — repository method name includes DeletedAtIsNull
Optional<T> findByIdAndDeletedAtIsNull(UUID id);
Page<T> findByStatusAndDeletedAtIsNull(String status, Pageable pageable);

// ❌ Wrong — may return soft-deleted records
Optional<T> findById(UUID id);
List<T> findAll();
```

**Verify soft delete in tests:**
```java
// After calling delete API — verify record is still in DB but marked
Entity entity = repository.findById(id).orElseThrow();
assertThat(entity.getDeletedAt()).isNotNull();      // ✅ deleted_at is set

// Verify deleted record not returned in list
mockMvc.perform(adminGet("/api/admin/<resource>"))
    .andExpect(jsonPath("$.data[?(@.id == '" + id + "')]").doesNotExist());
```

## Entity Conventions

```java
@Entity
@Table(name = "<table_name>")   // always explicit table name
@Getter
@Setter
@NoArgsConstructor              // required by JPA
public class <Name>Entity extends BaseEntity {

    // Domain fields — derived from requirements document
    @Column(nullable = false, length = 255)
    private String title;

    @Column(nullable = false, length = 255, unique = true)
    private String slug;

    @Column(columnDefinition = "TEXT")   // for long content — not VARCHAR
    private String content;

    // Enum field — always STRING, never ORDINAL
    @Column(nullable = false, length = 50)
    @Enumerated(EnumType.STRING)
    private <EnumType> status = <EnumType>.DEFAULT_VALUE;

    // Many-to-one relationship — always LAZY
    @ManyToOne(fetch = FetchType.LAZY)
    @JoinColumn(name = "<parent>_id")
    private <ParentEntity> parent;

    // One-to-many — always LAZY
    @OneToMany(mappedBy = "parent", fetch = FetchType.LAZY,
               cascade = CascadeType.ALL, orphanRemoval = true)
    @OrderBy("displayOrder ASC")
    private List<<ChildEntity>> items = new ArrayList<>();

    // Many-to-many — always LAZY, always declare join table
    @ManyToMany(fetch = FetchType.LAZY)
    @JoinTable(
        name = "<junction_table>",
        joinColumns = @JoinColumn(name = "<this>_id"),
        inverseJoinColumns = @JoinColumn(name = "<other>_id")
    )
    private List<<OtherEntity>> others = new ArrayList<>();
}
```

**Entity rules:**
```
✅ Extend BaseEntity — always
✅ @Table(name = "...") — always explicit
✅ FetchType.LAZY on all associations — never EAGER
✅ columnDefinition = "TEXT" for long content
✅ @Enumerated(EnumType.STRING) — never ORDINAL
✅ Initialize collections: = new ArrayList<>()
✅ @NoArgsConstructor — required by JPA
❌ Never @Data on JPA entity — causes Hibernate proxy issues
❌ Never FetchType.EAGER
❌ Never EnumType.ORDINAL
❌ Never put business logic inside entity methods
```

## Repository Conventions

```java
@Repository
public interface <Name>Repository extends JpaRepository<<Entity>, UUID> {

    // Single record — always include DeletedAtIsNull
    Optional<<Entity>> findByIdAndDeletedAtIsNull(UUID id);

    // Lookup by unique field
    Optional<<Entity>> findBy<Field>AndDeletedAtIsNull(<Type> value);

    // Existence check
    boolean existsBy<Field>AndDeletedAtIsNull(<Type> value);

    // Paged list — always Page<T>, never unbounded List<T>
    Page<<Entity>> findBy<Filter>AndDeletedAtIsNull(<Type> filter, Pageable pageable);

    // Bounded list — only for small, known-size results
    List<<Entity>> findTop<N>By<Filter>AndDeletedAtIsNullOrderBy<Field>Desc(...);

    // Count
    long countBy<Filter>AndDeletedAtIsNull(<Type> filter);
}
```

**Repository rules:**
```
✅ Every method includes DeletedAtIsNull — no exceptions
✅ Page<T> for all list endpoints — never unbounded List<T>
✅ List<T> only for bounded queries (top 3, featured items, etc.)
✅ Spring Data method naming preferred over @Query
✅ Use @Query only when method naming becomes unreadable
❌ Never findAll() without filters
❌ Never findById() without deleted_at check
❌ Never return soft-deleted records
```

## Slug Generation Pattern

When a design shows a URL-friendly identifier, generate it from the title:

```java
// Service layer — never user-entered, always auto-generated
private String generateSlug(String title) {
    return title.toLowerCase()
        .replaceAll("[^a-z0-9\\s-]", "")   // strip special chars
        .replaceAll("\\s+", "-")             // spaces → hyphens
        .replaceAll("-+", "-")               // collapse multiple hyphens
        .replaceAll("^-|-$", "")            // trim leading/trailing hyphens
        .trim();
}

// Handle collisions — if slug exists, append -1, -2, etc.
private String generateUniqueSlug(String title) {
    String base = generateSlug(title);
    String slug = base;
    int counter = 1;
    while (repository.existsBySlugAndDeletedAtIsNull(slug)) {
        slug = base + "-" + counter++;
    }
    return slug;
}
```

## Many-to-Many Junction Table Pattern

```sql
CREATE TABLE <a>_<b>_map (
    <a>_id  UUID NOT NULL REFERENCES <table_a>(id) ON DELETE CASCADE,
    <b>_id  UUID NOT NULL REFERENCES <table_b>(id) ON DELETE CASCADE,
    PRIMARY KEY (<a>_id, <b>_id)    -- composite PK, no separate id column
);

CREATE INDEX idx_<a>_<b>_map_<a> ON <a>_<b>_map(<a>_id);
CREATE INDEX idx_<a>_<b>_map_<b> ON <a>_<b>_map(<b>_id);
```

```java
@ManyToMany(fetch = FetchType.LAZY)
@JoinTable(
    name = "<a>_<b>_map",
    joinColumns = @JoinColumn(name = "<a>_id"),
    inverseJoinColumns = @JoinColumn(name = "<b>_id")
)
private List<<BEntity>> items = new ArrayList<>();
```

## Child Table Pattern (One-to-Many with Own Fields)

Use when a resource has sub-items that are too complex for a junction table:

```sql
CREATE TABLE <child_table> (
    id             UUID         PRIMARY KEY DEFAULT gen_random_uuid(),
    <parent>_id    UUID         NOT NULL REFERENCES <parent_table>(id),

    -- child-specific fields from requirements
    <field>        <TYPE>       NOT NULL,
    display_order  INT          NOT NULL DEFAULT 0,

    -- audit columns
    created_at     TIMESTAMPTZ  NOT NULL DEFAULT now(),
    updated_at     TIMESTAMPTZ  NOT NULL DEFAULT now(),
    deleted_at     TIMESTAMPTZ
);

CREATE INDEX idx_<child>_parent ON <child_table>(<parent>_id);
CREATE INDEX idx_<child>_active ON <child_table>(deleted_at)
    WHERE deleted_at IS NULL;
```

```java
@OneToMany(mappedBy = "<parent>", fetch = FetchType.LAZY,
           cascade = CascadeType.ALL, orphanRemoval = true)
@OrderBy("displayOrder ASC")
private List<<ChildEntity>> items = new ArrayList<>();
```

## Key-Value Settings Table Pattern

Use when the design shows a settings page with named key-value pairs:

```sql
CREATE TABLE <name>_settings (
    id          UUID         PRIMARY KEY DEFAULT gen_random_uuid(),
    key         VARCHAR(100) NOT NULL UNIQUE,
    value       TEXT,
    updated_at  TIMESTAMPTZ  NOT NULL DEFAULT now(),
    updated_by  UUID
);

CREATE INDEX idx_<name>_settings_key ON <name>_settings(key);

-- Seed the keys on migration — values can be NULL initially
INSERT INTO <name>_settings (key, value) VALUES
    ('<key_1>', NULL),
    ('<key_2>', NULL),
    ('<key_3>', NULL);
```

## Display Order / Reorder Pattern

When the design shows drag-and-drop ordering:

```sql
display_order INT NOT NULL DEFAULT 0

CREATE INDEX idx_<table>_order ON <table>(display_order)
    WHERE deleted_at IS NULL;
```

```java
// Query ordered list
List<T> findAllBy<Filter>AndDeletedAtIsNullOrderByDisplayOrderAsc();

// Reorder — service receives list of IDs in new order
@Transactional
public void reorder(List<UUID> orderedIds) {
    for (int i = 0; i < orderedIds.size(); i++) {
        final int position = i;
        repository.findByIdAndDeletedAtIsNull(orderedIds.get(i))
            .ifPresent(entity -> {
                entity.setDisplayOrder(position);
                repository.save(entity);
            });
    }
}
```

## ✅ Always Do This

1. Extend `BaseEntity` on every entity
2. Create a Flyway migration for every schema change
3. Include `deleted_at IS NULL` in every query method name
4. Use `TIMESTAMPTZ` in SQL — `Instant` in Java
5. Use `NUMERIC(19,4)` for money — `BigDecimal` in Java
6. Use `UUID` primary keys with `gen_random_uuid()`
7. Declare foreign keys explicitly in SQL and `@JoinColumn` in Java
8. Add index on every filtered, ordered, or joined column
9. Add the mandatory partial index: `WHERE deleted_at IS NULL`
10. Use `FetchType.LAZY` on all associations
11. Use `EnumType.STRING` — never `ORDINAL`
12. Initialize collections: `= new ArrayList<>()`
13. Generate slugs in service layer — never user-entered
14. Run `mvn flyway:validate` before marking any task complete
15. Write `DATABASE_AUDIT.md` entry for every curl operation

---

## ❌ Never Do This

1. Use `spring.jpa.hibernate.ddl-auto=update` or `create`
2. Modify an existing migration — create a new versioned file
3. Call `repository.delete()` or `deleteById()` — soft delete only
4. Query without `deleted_at IS NULL` — stale data leaks
5. Use `TIMESTAMP` — always `TIMESTAMPTZ`
6. Use `FLOAT` or `DOUBLE` — always `NUMERIC(19,4)` for decimals
7. Use sequential integer IDs — always UUID
8. Use `FetchType.EAGER`
9. Use `EnumType.ORDINAL`
10. Use `@Data` on JPA entities
11. Put business logic inside entity classes
12. Leave FK columns without an index
13. Skip `DATABASE_AUDIT.md` after curl tests
14. Use `LocalDateTime` — always `Instant` (UTC)
