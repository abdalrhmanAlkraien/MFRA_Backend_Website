## Business Description

The detail and edit view for a single testimonial. Accessed from the
Testimonials List (9.png) by clicking the pencil icon on any row.

Two purposes in one page:
1. **View full details** — shows the complete testimonial including the full
   quote, client photo, role, company, category, approval history, and
   current status
2. **Edit and update** — all fields are editable inline; changes auto-save

The page also contains the approval workflow controls:
- ADMIN can approve (Pending → Published) or reject (Pending → Draft)
- EDITOR can submit for approval (Draft → Pending)

This is the only place where the full untruncated testimonial quote is visible
(the list at 9.png shows only an excerpt).

## User Access

- [ ] PUBLIC — not accessible
- [x] ADMIN — full access — view, edit, approve, reject, publish, delete
- [x] EDITOR — view and edit — can submit for approval, cannot approve/publish
  directly, cannot delete

> Unauthenticated → redirect to 1.png (Admin Login).

## Permissions

**ADMIN can:**
- View all fields (full quote, client details, category, status, history)
- Edit all fields
- Change status: any transition
    - Draft → Pending, Draft → Published
    - Pending → Published (approve), Pending → Draft (reject with note)
    - Published → Draft (unpublish/retract)
- Delete testimonial (with confirmation) — navigates back to 9.png after delete
- Upload / replace client photo → S3
- Set display order (position on the public testimonials section)

**EDITOR can:**
- View all fields
- Edit all content fields (quote, client name, role, company, category, photo)
- Submit for approval: Draft → Pending
- Cannot approve or publish directly
- Cannot reject
- Cannot delete
- Cannot change display order

Create: None — this is edit only (creation is on 11.png)
Edit:   PATCH /api/admin/testimonials/:id
Delete: DELETE /api/admin/testimonials/:id (ADMIN only)

- Required

**JOB TITLE field:**
- Label: "JOB TITLE" (label font, uppercase, muted)
- Input: text input, max 100 chars

**COMPANY field:**
- Label: "COMPANY" (label font, uppercase, muted)

**CATEGORY field:**

### Right Panel — Status and Workflow

**CURRENT STATUS section:**
- Label: "CURRENT STATUS" (label font, uppercase, muted)
- Large status badge (same style as 9.md table badges but larger):
    - DRAFT: surface_container_highest bg + on_surface_variant text
    - PENDING: primary_container/20 bg + primary_container text
    - PUBLISHED: tertiary_container/20 bg + tertiary_container text
- Status badge is display only here — changed via the action buttons below

**WORKFLOW ACTION BUTTONS:**

Buttons shown depend on current status and role:

```
Current: DRAFT
  EDITOR: [Submit for Approval]   ← Pending button, primary outline
  ADMIN:  [Submit for Approval]   [Publish Directly]

Current: PENDING
  EDITOR: [Withdraw Submission]   ← reverts to Draft
  ADMIN:  [Approve & Publish]     [Reject — Return to Draft]

Current: PUBLISHED
  EDITOR: (no workflow actions — read-only for published)
  ADMIN:  [Unpublish — Return to Draft]
```

Button styles:
- Positive actions (Approve, Publish): primary gradient
- Neutral actions (Submit, Withdraw): secondary style
- Negative/destructive actions (Reject, Unpublish): error color ghost border

**REJECT REASON field (ADMIN, PENDING status only):**
- Shown when ADMIN clicks "Reject — Return to Draft"
- Appears as an inline textarea before the rejection is confirmed
- Placeholder: "Provide a reason for rejection (optional)"
- On submit: status → DRAFT, rejection note saved to activity log

- "Remove photo" — tertiary text link, error color, small

### API Endpoints

```
GET /api/admin/testimonials/:id
  Response: {
    id:               UUID
    client_name:      string
    client_photo_url: string | null
    job_title:        string
    company:          string
    category:         string
    quote:            string  (full, untruncated)
    status:           DRAFT | PENDING | PUBLISHED
    display_order:    number
    created_at:       timestamp
    updated_at:       timestamp
    activity_log:     array of log entries
  }

PATCH /api/admin/testimonials/:id
  → Auto-save and field updates (partial)
  Request: any subset of editable fields

PATCH /api/admin/testimonials/:id/submit
  → EDITOR or ADMIN: Draft → Pending
  → No request body

PATCH /api/admin/testimonials/:id/withdraw
  → EDITOR or ADMIN: Pending → Draft
  → No request body

PATCH /api/admin/testimonials/:id/approve
  → ADMIN only: Pending → Published
  → No request body

PATCH /api/admin/testimonials/:id/reject
  → ADMIN only: Pending → Draft
  Request: { reason: string (optional) }

PATCH /api/admin/testimonials/:id/publish
  → ADMIN only: Draft → Published (bypass pending)
  → No request body

PATCH /api/admin/testimonials/:id/unpublish
  → ADMIN only: Published → Draft
  → No request body

DELETE /api/admin/testimonials/:id
  → ADMIN only — soft delete

POST /api/admin/upload/presigned-url
  Request: { filename, content_type, folder: "testimonials/photos" }
  Response: { presigned_url, public_url, key }
```

### Business Rules

- All status transitions enforced at backend — not just hidden in UI
- EDITOR cannot call approve, reject, publish, unpublish endpoints → 403
- Rejection note is optional but stored in activity log when provided
- Display order change takes effect immediately on the public site
  (cache invalidated on save)
- Only PUBLISHED testimonials appear on the public website
- Activity log is append-only — no edits or deletes
- Auto-save saves content fields — status changes are explicit button actions
- Character limit on quote: 500 characters (enforced client and server side)
- Client photo uploads to `testimonials/photos/` folder on S3
- Photo is optional — testimonial can be published without a photo
- Display order defaults to the next available number on creation
  (e.g. if 42 testimonials exist, default display_order = 43)
- Changing display order of one testimonial does not reorder others
  (each testimonial has its own order number — gaps are allowed)

---

### Edge Cases

- Testimonial not found (deleted by another admin) → redirect to
  /admin/testimonials with toast: "Testimonial no longer exists"
- Concurrent editing (two admins editing same testimonial) → last write wins
  (no conflict detection in MVP — note in UI: "Auto-saving...")
- Photo upload fails → error shown on avatar slot, testimonial still saves
- Quote exactly 500 chars → allowed, character counter shows "500 / 500" in amber
- Quote over 500 chars → input blocks further typing, counter shows red
- Rejection without reason → allowed (reason field optional)
- EDITOR tries to call approve endpoint directly → 403 from backend
- Status change while auto-save is in flight → auto-save completes first,
  then status change applies
- Empty activity log (brand new testimonial) → "No activity yet" in the log section
- Display order conflict (two testimonials same number) → both show, public
  site orders alphabetically when order numbers match

### Out of Scope

- Bulk editing multiple testimonials at once
- Testimonial versioning or revision history (activity log only, not content diffs)
- Sending an email notification to the EDITOR when their testimonial is approved or rejected
- Scheduling a publish date/time for a testimonial
- Duplicating a testimonial
- Embedding the testimonial in a specific page (managed via display order only)
- Preview in a new browser tab showing the public website view
  (preview modal on 9.md covers basic preview needs)
- Star rating or scoring system
- Testimonial expiry date
