# Admin Login Page

## Business Description

The entry point to the MFRA Cloud Intelligence Admin Control Center.
Authenticated admin users and editors log in here using email and password.
On successful login the user is redirected to the Admin Dashboard (2.md).
If the user is already authenticated they are redirected to the dashboard
immediately — the login page is never shown to a logged-in user.

This is a security-critical page. Failed login attempts are tracked per IP
address. After multiple failures the IP is blocked. All access attempts are
logged for audit purposes.

## User Access

- [x] PUBLIC — accessible by anyone who is not already logged in
- [ ] ADMIN — if already logged in, redirect to dashboard
- [ ] EDITOR — if already logged in, redirect to dashboard

> This page is the only public page inside the /admin path.
> All other /admin/* routes redirect here when unauthenticated.

## Permissions

View:
- Login form (email, password, remember device checkbox)
- Forgot Password link
- Security notice text
- Support and System Status links in footer

Create:
- Submit login credentials → POST /api/auth/login
- (Future) Request password reset → POST /api/auth/forgot-password

Edit:   None
Delete: None

Other:
- Toggle password visibility (show/hide — frontend only, no API call)
- Remember this device — extends JWT token TTL when checked
- Redirect to /admin/dashboard on successful login
- Redirect away from /admin/login if already authenticated

## Related Pages

- 2.md (Admin Dashboard) → successful login redirects here
- All /admin/* pages → redirect to this page when JWT is missing or expired

## Notes

**Branding:**
- Logo: orange cloud checkmark icon + "MFRA Cloud Intelligence" in orange
- Subtitle: "ADMIN CONTROL CENTER" in muted uppercase letter-spacing
- Dark navy background (#0d1117 range) — not plain black
- Orange (#F59E0B range) as the primary accent color for buttons and logo

**Form behavior:**
- Email field: placeholder "admin@mfra.cloud"
- Password field: show/hide toggle with eye icon on the right
- "Forgot Password?" — right-aligned link above the password field
- "Remember this device" checkbox — when checked, refresh token TTL extends
  to 30 days instead of the default 7 days
- Submit button text: "Login to Console →" with right-arrow icon
- Submit button: full-width, orange background, dark text

**Security notice (visible below the button):**
- Shield icon + text: "Encrypted with 256-bit SSL. Unauthorized access
  attempts to the MFRA Cloud Intelligence environment are strictly monitored
  and logged. IP addresses will be blacklisted upon multiple failed attempts."
- This is static display text — not dynamic

**Error states:**
- Invalid credentials → inline error below the button (not a page redirect)
  Message: "Invalid email or password. Please try again."
- Account locked (IP blocked) → error: "Access temporarily restricted.
  Contact your system administrator."
- Network error → error: "Unable to connect. Please check your connection."
- All errors shown inline in the card — never toast or alert dialog

**Loading state:**
- Button shows spinner and "Authenticating..." while the API call is in flight
- Inputs are disabled during submission

**Footer:**
- Left: "© 2024 MFRA CLOUD INTELLIGENCE. BUILT FOR AWS ECOSYSTEM."
- Right: SUPPORT link, SYSTEM STATUS link
- Both links are static — no API calls needed

**Responsive behavior:**
- Desktop (1280px+): centered card, max-width ~420px, full dark background
- Tablet (768px–1279px): same layout, card fills ~90% width
- Mobile (< 768px): card fills full width with 16px horizontal padding,
  logo and subtitle remain centered, footer stacks vertically

**Security rules (backend):**
- Max 5 failed attempts per IP per 15 minutes
- On 6th failure: block IP for 1 hour, log the event
- All login attempts (success and failure) logged with IP, timestamp, email
- JWT access token: 1 hour TTL
- JWT refresh token: 7 days (30 days if "Remember this device" is checked)
- Blacklisted JWTs stored in Redis on logout

**Out of scope:**
- Social login (Google, GitHub, etc.)
- Two-factor authentication (2FA) — deferred
- Registration page — admins are created by other admins only
- CAPTCHA — not in MVP
- Password strength meter (login only, not registration)
- "System Status" and "Support" links do not need real endpoints in MVP —
  they can link to external URLs defined in settings
