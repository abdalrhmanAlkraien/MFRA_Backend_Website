# Blog Post Editor

## Business Description

The rich content editor for creating and editing blog posts in the MFRA admin
panel. Accessed from the Blog List (3.png) either by clicking "New Post" or
by clicking "Edit" on an existing article row.

Two states — same page, same URL pattern:
- **Create mode:** `/admin/blogs/new` — all fields empty
- **Edit mode:** `/admin/blogs/:id/edit` — fields pre-populated from existing article

The editor is split into two panels: the main writing area on the left (title,
content type, category, rich text body) and the publishing settings on the right
(status, featured toggle, display priority, metadata, SEO preview).

Auto-save runs every 60 seconds and on each field change with a debounce.
Images (hero and gallery) upload directly to AWS S3 via pre-signed URLs —
no image data passes through the backend API server.

The "Publish to Site" button changes the article status to PUBLISHED and
makes it visible on the public website immediately.

## User Access

- [ ] PUBLIC — not accessible
- [x] ADMIN — full access — can create, edit, publish, feature, set priority
- [x] EDITOR — full access to content fields — restricted on some settings
  (see Permissions below)

> Unauthenticated → redirect to 1.png (Admin Login).

## Permissions

**ADMIN can:**
- Edit all fields (title, content, category, content type)
- Set Publishing Status (Draft / Published / Archived)
- Toggle Featured Post on/off
- Set Display Priority (1–10)
- Add and remove tags/categories in Post Metadata
- Change Author assignment
- Upload, replace, and delete Hero image
- Upload, add, and delete Gallery images
- Discard / delete the article
- Publish to Site

**EDITOR can:**
- Edit all content fields (title, content, category, content type)
- Set Publishing Status (Draft / Published only — cannot Archive)
- Toggle Featured Post — NOT available (hidden for EDITOR)
- Display Priority — NOT available (hidden for EDITOR)
- Add tags/categories in Post Metadata
- Cannot change Author assignment (locked to their own account)
- Upload Hero and Gallery images
- Publish to Site

Create: POST /api/admin/blogs (on first save)
Edit:   PUT  /api/admin/blogs/:id
Delete: Triggered by "Discard" → DELETE /api/admin/blogs/:id (with confirmation modal)

Other:
- Auto-save (PATCH /api/admin/blogs/:id every 60s or on debounced change)
- Upload image to S3 via pre-signed URL
- Publish to Site → PATCH /api/admin/blogs/:id/publish
- Discard → DELETE /api/admin/blogs/:id with confirmation modal

**Rich Text Editor Area:**
- Placeholder: "Write your technical insight here using Markdown..."
- Background: surface_container_lowest (recessed / inset feel)

- TipTap renders HTML but accepts Markdown shortcuts

**Auto-save Status Bar (below the editor, above the action buttons):**
- Left side:

**Auto-save behavior:**
- Debounce: saves 3 seconds after user stops typing
- Minimum interval: no more than one auto-save per 60 seconds
- Save indicator: brief "Saving..." text replaces "Auto-save enabled" during save
- On save success: returns to "Auto-save enabled" + updates "Last change at HH:MM"
- On save failure: shows "Save failed — retry" in red, manual save needed
- Create mode: first auto-save creates the record (POST), subsequent auto-saves update (PATCH)

### Media Assets Section (below the editor, left panel)

**Section header:**
- Image/gallery icon + "Media Assets" title (headline font)
- "Upload New" button — secondary style with upload icon

**Image cards (horizontal row, 3 slots visible):**

**Slot 1 — Hero Image:**
- Shows uploaded image thumbnail
- "HERO" badge overlay — bottom-left, orange pill
- Edit icon (pencil) — replaces the hero image
- Delete icon (trash) — removes the hero image
- On click edit: opens file picker → uploads to S3 → shows new thumbnail

**Slot 2 — Add Gallery:**
- Empty state slot with "+" icon and "ADD GALLERY" label
- Clicking opens file picker for additional gallery images
- After upload: slot transforms into an image card (same style as Hero)

**Slot 3 — Uploading state:**
- Shows upload progress: spinning icon + "Uploading... 74%"
- Progress percentage updates in real time
- After completion: transforms into image card with the uploaded image

**S3 Upload Flow:**
```
1. User selects file from file picker
2. Frontend calls POST /api/admin/upload/presigned-url
   → Backend generates S3 pre-signed URL (15 min expiry)
   → Returns: { presigned_url, public_url, key }
3. Frontend uploads file directly to S3 using presigned_url
   → No image data passes through backend server
4. On S3 upload success: save public_url in article data
   → PATCH /api/admin/blogs/:id with { hero_image_url: public_url }
5. Show uploaded thumbnail immediately using public_url
```

**File constraints:**
- Accepted formats: image/jpeg, image/png, image/webp
- Maximum size: 5 MB per file
- Minimum dimensions for Hero: 800×400px (show warning if smaller)
- No minimum for gallery images

**Upload states to handle:**
- Selecting: file picker opens
- Validating: check format + size client-side before upload
- Uploading: progress bar / percentage shown in the card slot
- Success: show thumbnail, remove progress indicator
- Error: show red error state in the card slot with retry option
  "Upload failed — tap to retry"

### API Endpoints

```
GET  /api/admin/blogs/:id
  → Load existing article data for edit mode

POST /api/admin/blogs
  → Create new article (triggered by first auto-save in create mode)
  Request: { title, content, content_type, category_id, status }

PATCH /api/admin/blogs/:id
  → Auto-save and manual field updates
  Request: partial — only changed fields

PUT /api/admin/blogs/:id
  → Full update on explicit save

PATCH /api/admin/blogs/:id/publish
  → Triggered by "Publish to Site" button
  → Sets status = PUBLISHED, sets published_at = now()

DELETE /api/admin/blogs/:id
  → Triggered by "Discard" after confirmation

GET /api/admin/blog-categories
  → Load category options for the Category dropdown

GET /api/admin/blog-tags
  → Load existing tags for autocomplete

POST /api/admin/upload/presigned-url
  Request:  { filename, content_type, folder: "blogs" }
  Response: { presigned_url, public_url, key }
  → Backend generates S3 pre-signed URL
  → Frontend uploads directly to S3

PATCH /api/admin/blogs/:id
  → After S3 upload success: { hero_image_url } or { gallery_image_urls }
```

### Business Rules

- Title is required before first publish — show inline error if empty on publish attempt
- Content is required before publish — show inline error if empty
- Category is required before publish — show inline error if empty
- Auto-save works even when title/content are empty (saves draft state)
- Only one FEATURED post at a time — turning on warns and unfeatures previous
- Slug auto-generated from title: lowercase, hyphens, URL-safe
  → If slug already exists → append -2, -3 etc. (handled by backend)
- Display Priority 1 = highest position, 10 = lowest
- EDITOR cannot set status to Archived
- EDITOR cannot see or change Featured Post or Display Priority
- EDITOR cannot change the Author field — locked to themselves
- Images upload to S3 directly via pre-signed URL — never through the backend
- Hero image is optional — article can be published without it
- Maximum 10 gallery images per article
- All times in auto-save bar shown in local time of the admin user's browser

---

### Edge Cases

- Browser tab closed mid-edit → auto-save preserves last saved state
- S3 upload fails → show error state in card slot, article still saves (just without the image)
- Featured post conflict → confirmation modal before proceeding
- Publish with empty title → inline error on title field, no API call made
- Publish with empty content → inline error on content, no API call made
- Category not selected → inline error on category dropdown
- Very long title → slug truncated at 100 characters
- Network offline during auto-save → show "Save failed" with retry
- Edit mode — article not found (deleted by another admin) → redirect to /admin/blogs with toast
- File too large (> 5MB) → client-side validation error before upload starts
- File wrong format → client-side validation error before upload starts
- Image below minimum dimensions → warning shown but upload still allowed

---

### Out of Scope

- Version history / revision tracking
- Collaborative editing (multiple users editing same post simultaneously)
- Scheduled publishing (set a future publish date/time)
- Post preview in a new tab (showing how the public will see it)
- Custom slug editing (auto-generated only in MVP)
- SEO meta title / meta description manual override
- Social sharing image (OG image) separate from hero image
- Video upload (images only in MVP)
- Comment moderation from the editor
- Duplicate / clone post action
- Markdown raw mode toggle (TipTap only in MVP)
- Keyboard shortcuts documentation panel
