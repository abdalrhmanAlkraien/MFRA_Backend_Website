# User Intelligence — User Details Page

## Business Description

A read-only intelligence profile for a single admin or editor account.
Accessed from the Users Management list (12.png) by clicking a user row
or the eye icon.

Titled "User Intelligence" — consistent with the platform's analytical
branding. Shows three panels: identity card (who the person is), access
level (what permissions they have), and system activity (how they
contribute to the platform).

Two primary actions are available: Edit Profile (→ 14.png) and
Deactivate Account. Both are immediate actions the viewing admin can
take without navigating elsewhere.

## User Access

- [ ] PUBLIC — not accessible
- [x] ADMIN — full access — view details, edit, deactivate
- [ ] EDITOR — no access — page hidden entirely

> EDITOR cannot access user management at all.
> Unauthenticated → redirect to 1.png.

## Permissions

**ADMIN can:**
- View all user details (identity, permissions, activity)
- Click "Edit Profile" → navigates to 14.png
- Click "Deactivate Account" → changes status Active → Inactive
  (if user is already Inactive: button shows "Reactivate Account")
- View permission tags (WRITE_ALL, DELETE_SECURE, GLOBAL_EXEC)
- View system activity metrics (uptime, ticket resolution)

**EDITOR:** No access.

### API Endpoints

```
GET /api/admin/users/:id
  Response: {
    id:               UUID
    full_name:        string
    job_title:        string  (the display title, e.g. "Senior Cloud Editor")
    email:            string
    role:             ADMIN | EDITOR
    status:           ACTIVE | INACTIVE
    photo_url:        string | null
    created_at:       timestamp
    last_login:       timestamp | null
    permission_tags:  string[]  (e.g. ["WRITE_ALL", "DELETE_SECURE"])
    access_description: string  (e.g. "Current permission tier set to Root...")
    system_activity: {
      uptime_contribution: number  (percentage, e.g. 99.2)
      ticket_resolution_avg_hours: number  (e.g. 4.2)
    }
  }

PATCH /api/admin/users/:id
  Request: { status: INACTIVE | ACTIVE }
  → Deactivate or reactivate
```

### Business Rules

- ADMIN only — EDITOR cannot access
- "Deactivate Account" button hidden when viewing own profile
- "Deactivate" → "Reactivate" when user is already INACTIVE
- Permission tags are derived from role — not per-user configurable in MVP
- Job title (display title) is a separate field from the system role
- System activity metrics are read from a monitoring/audit system
- Uptime Contribution is display-only — cannot be edited

---

### Edge Cases

- User not found → redirect to /admin/users with toast "User not found"
- Viewing your own profile → Deactivate button hidden
- User already inactive → button shows "Reactivate Account"
- Zero permission tags → show "No special permissions" in the tags area
- Uptime = 0% → show "0%" with empty progress bar
- No system activity data yet → show "No activity data" in the panel

---

### Out of Scope

- Editing directly from this page (Edit button navigates to 14.png)
- Viewing content created by this user
- Resetting the user's password from this page
- Viewing full audit log (separate page, not in MVP)
- Sending a message to the user from this page
