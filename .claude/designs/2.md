# Admin Dashboard — Global Intelligence Overview

## Business Description

The first page an authenticated admin user sees after login. It is the central
command view of the MFRA Cloud Intelligence platform — showing real-time system
health, user distribution, recent activity logs, server performance metrics, and
security status at a glance.

The dashboard gives administrators an immediate situational awareness snapshot
without needing to navigate to any other section. It is read-only — no create,
edit, or delete actions happen here. The "Refresh Data" button forces a fresh
API call to update all panels simultaneously.

This page also defines the global layout shell — the fixed left sidebar and
the top navigation bar — that is shared across all admin pages (3.png, 6.png,
9.png, 11.png, 14.png). The layout shell is implemented once and reused.

## User Access

- [ ] PUBLIC — not accessible, redirect to login
- [x] ADMIN — full access, all panels visible

### API Endpoints

```
GET /api/admin/dashboard/summary
  Response:
    total_users:    number
    active_admins:  number
    active_editors: number
    users_trend:    string ("+12%")
    admins_status:  string ("System Peak")
    editors_status: string ("Stable")

GET /api/admin/dashboard/access-logs?limit=10
  Response: array of access log entries
    id, user_name, action, location_path,
    ip_address, geo_location, event_type, created_at

GET /api/admin/dashboard/node-performance
  Response:
    cpu_usage:     number (percentage 0–100)
    memory_load:   number (percentage 0–100)
    storage:       number (percentage 0–100)

GET /api/admin/dashboard/security-status
  Response:
    condition:     GREEN | YELLOW | RED
    status_text:   string
    last_audit_at: timestamp
    link_established: boolean
```

All 4 endpoints are called in parallel on page load and on "Refresh Data" click.
Polling interval: none (manual refresh only).

### Business Rules

- JWT must be valid to access this page — expired JWT → redirect to 1.png
- EDITOR role sees masked user names in access logs ("User ***")
- Node Performance and Security Status are read-only for all roles
- "View All Logs" link is non-functional in MVP — shows as disabled
- Global command search is non-functional in MVP — visual placeholder only
- Notification bell has no badge and no dropdown in MVP
- All 4 API calls happen in parallel — page does not wait for one before starting another
- If any single API call fails, that panel shows an error state — other panels still load
- "Refresh Data" re-calls all 4 endpoints simultaneously
- Last audit timestamp is clickable and shows the exact datetime on hover (tooltip)

---

### Edge Cases

- If total_users = 0 → show "0" not blank
- If access_logs returns empty → show "No recent activity" empty state in the panel
- If node performance > 90% on any metric → show red color + warning icon next to the metric
- If security condition = RED → entire Security Status panel border turns red (ghost border exception)
- If JWT expires while on dashboard → silent redirect to login without alert
- Loading state: skeleton placeholders for all 3 stat cards and both panels
  Use surface_container_highest as skeleton background with shimmer animation

---

### Out of Scope

- Global command search functionality (visual placeholder only)
- Notification system (bell icon present but non-functional)
- Real-time WebSocket updates (manual refresh only for MVP)
- "View All Logs" page (link present but non-functional)
- Clicking on individual log entries (no detail view in MVP)
- Downloading or exporting dashboard data
- Date range filtering on access logs
- Historical charts or trend graphs (numbers only, no charting library)
- Dark/light mode toggle (always dark per DESIGN.md)
