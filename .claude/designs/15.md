# Add New User Account Page

## Business Description

The form for provisioning a new admin or editor account into the MFRA
Cloud Intelligence platform. Accessed from the Users Management list
(12.png) via the "+ Add New User" button.

Key design decision: when a new user is created, the system automatically
sends them an invitation email with temporary credentials. The admin does
NOT set the user's password here — the system generates a temporary one
and forces a password change on first login.

The page uses a two-column layout: left side provides context and security
information (description, RBAC security note, quota usage), right side
contains the actual creation form.

Role selection uses a card-based selector (not a dropdown) — two cards
side by side with icons and descriptions, one selected with an orange
border. This is a more deliberate UX than a dropdown, reinforcing the
weight of the access decision.

**Top bar branding:** Same as 14.png ("MFRA Intelligence / CLOUD CONTROL
CENTER"). "System Online" + "Admin User" shown instead of role badge.
Bottom-left sidebar shows "QUOTA USAGE: 650/1000 Cloud Seats".

### Success Flow

```
Admin fills form → clicks "Save User"
  ↓
POST /api/admin/users { full_name, email, role, status: ACTIVE }
  ↓
Backend:
  1. Creates admin_users record
  2. Generates temporary password (random, 16 chars)
  3. Sets force_password_change = true
  4. Sends invitation email to the new user's email address
     Subject: "You have been added to MFRA Cloud Intelligence"
     Body: temporary credentials + login URL + password change requirement
  ↓
Frontend:
  Redirect to /admin/users
  Toast: "Account created. Invitation email sent to [email]."
```

### API Endpoints

```
GET /api/admin/users/quota
  → Returns { used: 650, total: 1000 }
  → Called on page load to populate quota display

POST /api/admin/users
  Request: {
    full_name: string (required)
    email:     string (required, unique)
    role:      ADMIN | EDITOR (required)
    status:    ACTIVE (always active on creation)
  }
  Response: {
    id:     UUID
    email:  string
    role:   string
    status: string
  }
  Errors:
    409 — email already in use
    422 — validation errors
    403 — quota reached
```

### Business Rules

- Admin does NOT set the new user's password — system generates temp password
- New user receives invitation email automatically on account creation
- New user MUST change their password on first login (force_password_change)
- MFA is mandatory for all new accounts — not configurable at creation
- Session timeout is 8 hours for all users — not configurable at creation
- Email must be unique across all admin_users
- Quota limit enforced — cannot create user when at 1000 Cloud Seats
- New users are always created as ACTIVE status
- EDITOR cannot access this page — enforced at both UI and API level
- Role cannot be changed from this page after creation
  (role change done via 14.md Edit Profile → actually requires a separate
  role-change endpoint, not covered in this page)

---

### Edge Cases

- Email already exists → inline error: "An account with this email already exists"
- Quota full → Save button disabled + warning shown above button
- Invitation email fails to send → account still created, toast shows:
  "Account created but invitation email failed. Share credentials manually."
- Long full name → validation at 100 chars
- Invalid email format → inline error: "Please enter a valid email address"
- Both role cards somehow unselected (edge case) → Save button disabled
    + inline error: "Please select an access role"
- Network error on POST → toast error, form state preserved

### Out of Scope

- Setting a custom password for the new user (temp password only)
- Adding the user to groups or teams
- Setting per-user permissions (role-based only in MVP)
- Configuring MFA method at creation (always required, method set on first login)
- Setting a custom session timeout per user
- Uploading a profile photo at creation (done later via Edit Profile, 14.png)
- Sending a custom invitation message
- Bulk user import from CSV
- Creating users via API key (admin UI only in MVP)
