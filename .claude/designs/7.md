# Case Study Insights Page

## Business Description

A per-case-study analytics dashboard showing detailed performance metrics
for a single published case study. Accessed from the Case Studies List (6.png)
by clicking the "Insights" action on a published row.

Unlike the Blog Article Insights (4.png), this page is focused on lead
generation metrics: consultation clicks, conversion rate, and authenticated vs
guest visitor tracking. Case studies attract decision-makers considering MFRA
services — understanding who reads them and how they engage is critical for
the sales team.

Two additional actions distinguish this from the blog insights page:
- **Export PDF** — generates a shareable PDF report of all metrics
- **Share Report** — generates a shareable link to this insights view

The page uses a bar chart (not line chart) for engagement trends because
case study traffic is more episodic and spiky — bar charts communicate
bursts of interest better than smooth lines.

### API Endpoints

```
GET /api/admin/case-studies/:id/insights/summary
  Response:
    case_study_id:          UUID
    title:                  string
    slug:                   string
    total_views:            number
    views_trend_percent:    number (+14.2)
    avg_read_time_sec:      number (272 = 4m 32s)
    read_time_lift_sec:     number (45 = +0m 45s)
    consultation_clicks:    number
    consultation_ctr_growth: number (+28)
    conversion_rate:        number (6.74)
    performance_label:      HIGH_PERFORMING | STANDARD | NEEDS_ATTENTION

GET /api/admin/case-studies/:id/insights/engagement?period=daily
  Query params: period = daily | weekly | monthly
  Response: array of { date: string, views: number }

GET /api/admin/case-studies/:id/insights/geo
  Response:
    map_data: array of { lat, lng, city, country, views }
    countries: array of { country_name, percentage }
                sorted by percentage desc, max 5

GET /api/admin/case-studies/:id/insights/access-log?limit=10
  Response: array of {
    ip_address:  string
    city:        string
    country:     string
    continent:   AMERICAS | EUROPE | ASIA_PACIFIC | MENA
    status:      AUTHENTICATED | GUEST
    os:          string
    browser:     string
    browser_version: string
    accessed_at: timestamp
  }

POST /api/admin/case-studies/:id/insights/export-pdf
  Response: application/pdf binary
  → Generates PDF with all current metrics

POST /api/admin/case-studies/:id/insights/share
  Response: { share_url: string, expires_at: timestamp }
  → Creates a time-limited shareable link (7 days)
```

### Business Rules

- Page only accessible for PUBLISHED case studies
  → DRAFT or ARCHIVED → redirect to /admin/case-studies with toast:
  "Insights are only available for published case studies"
- Case study not found → redirect to /admin/case-studies with 404 toast
- Total views shown as full integer — no "k" abbreviation (unlike 4.md)
  because exact numbers matter for sales reporting
- Avg read time format: `Xm Ys` (not `X:Y min` as in 4.md)
- Performance labels for conversion rate:
    - > 5%: HIGH PERFORMING (primary badge)
    - 2–5%: no badge, show trend vs last period
    - < 2%: NEEDS ATTENTION (error badge)
- Bar chart defaults to Daily on load
- Geographic reach shows top 5 countries maximum
- Access log shows 10 most recent entries
- Consultation clicks tracked when visitor clicks any CTA linking to
  /free-consultation from this case study's public page
- AUTHENTICATED status means the visitor had a valid session cookie —
  they are a returning registered user (if user accounts exist in future)
  In MVP: tracking is cookie-based, not tied to actual user accounts
- Share Report link expires after 7 days — backend enforces this
- Export PDF is generated server-side on demand — not pre-cached
- All times displayed in the admin's local browser timezone

### Edge Cases

- Zero views (newly published) → show "0" for all metrics, empty chart
  with "No data yet" overlay, empty geo panel, empty access log
- Export PDF takes > 5 seconds → button stays in "Generating..." state
  with spinner until download begins
- Share link copy fails (clipboard not available) → show the URL in a
  modal so user can copy manually
- Engagement chart has no data for selected period → show empty chart
  with "No data for this period" centered in the chart area
- Geographic data not available → show map without dots, message
  "Geographic data not yet available" below the map
- Access log empty → show "No access logs yet" in the table area
- Conversion rate exactly 5% → HIGH PERFORMING (threshold is ≥ 5%)
- Very long case study title → truncate breadcrumb at 30 characters with
  ellipsis, full title shown in page heading without truncation
