# Blog Article Insights Page

## Business Description

A per-article analytics dashboard showing detailed performance metrics for
a single published blog post. Accessed by clicking an "Insights" or "Analytics"
action on a blog row in the Blog List page (3.png).

The page title format is: "Article Insights: [Article Title in orange]"
giving the admin immediate context of which article they are analyzing.

Shows four performance KPIs, geographic distribution of readers on an
interactive world map with a data table, a real-time access log of recent
readers by IP and device, and an engagement trend line chart over a
selectable time period.

This is a read-only analytics page. No editing, creating, or deleting happens
here. It exists to help admins understand how individual articles are performing
and where their audience comes from.

### URL Structure

```
/admin/blogs/:blogId/insights
```

- `blogId` is the UUID of the blog article
- Navigated to from 3.png Blog List via "Insights" action button on each row
- "← BACK TO ARTICLES" link returns to /admin/blogs

---

### API Endpoints

```
GET /api/admin/blogs/:id/insights/summary
  Response:
    blog_id:          UUID
    title:            string
    total_views:      number
    views_trend:      string ("+18% this week")
    avg_read_time_sec: number (converted to mm:ss in frontend)
    avg_read_time_industry_sec: number (industry benchmark)
    conversion_rate:  number (percentage)
    bounce_rate:      number (percentage)
    bounce_rate_trend: string ("-2% from last post")

GET /api/admin/blogs/:id/insights/geo?view=world
  Query params: view = world | regions
  Response:
    map_data:  array of { lat, lng, city, country, views }
    table_data: array of { country_code, country_name, views, percentage }
    total_countries: number

GET /api/admin/blogs/:id/insights/access-log?limit=8
  Response: array of {
    ip_address:  string
    city:        string
    country:     string
    browser:     string
    os:          string
    accessed_at: timestamp
  }

GET /api/admin/blogs/:id/insights/engagement?period=7d
  Query params: period = 7d | 30d | 90d
  Response: array of {
    date:  string (ISO)
    views: number
  }
```

### Business Rules

- Page is only accessible if the blog exists and is published or was published
  → if blog is DRAFT → redirect to /admin/blogs with a toast: "Insights are
  only available for published articles"
- `avg_read_time` displayed as mm:ss format (e.g. 405 seconds → "6:45 min")
- `total_views` abbreviated: < 1000 = full number, ≥ 1000 = "k" format
- Bounce rate trend: downward = positive (green), upward = negative (red)
  (opposite of total views where upward = positive)
- Country table shows top 5 countries — no pagination in MVP
- Access log shows the 8 most recent entries — no pagination in MVP
- Geographic map defaults to "World" view on load
- Engagement chart defaults to "Last 7 Days" on load
- All four summary stats load from a single API call for performance
- Geo data and chart data are separate API calls (different time ranges)
- All times shown in relative format ("2m ago") except the system status
  footer which shows UTC absolute time

---

### Edge Cases

- Blog not found (404) → redirect to /admin/blogs with toast
- Blog is DRAFT → redirect to /admin/blogs with toast
- Zero views → show "0" for total views, hide trend line, show empty state
  in chart with "No data for this period"
- No geographic data → show world map without dots, empty country table
  with "No geographic data yet" message
- Access log empty → show "No readers logged yet" in the panel
- Chart period returns no data → show empty chart with period label
- Very long article title → truncate with ellipsis after 80 characters in
  the page header (full title in browser tab / tooltip on hover)
- Numbers > 1M → show "1.2M" format

---

### Out of Scope

- Exporting analytics as CSV or PDF
- Comparing two articles side by side
- Filtering access log by country or browser
- Sorting the country table (clicks on column headers do nothing)
- User-level tracking (IP only — no individual user profiles)
- Real-time WebSocket updates (page data loaded on mount and on refresh)
- Editing the article from this page (no edit button)
- Deleting the article from this page
- Comments or engagement per section of the article
- Heatmaps or scroll depth analytics
- A/B testing metrics
