# Edit Profile Page

## Business Description

The account editing interface for updating a user's profile information
and security credentials. Accessed from the User Details page (13.png)
via the "Edit Profile" button, or directly from the Users list (12.png)
via the pencil icon on a row.

**Important branding distinction from 13.png:** This page uses slightly
different branding — the top bar shows "MFRA Intelligence / CLOUD CONTROL
CENTER" (not "MFRA Admin / CLOUD INTELLIGENCE"). The search placeholder
is "Search system logs..." and a "SYSTEM ONLINE" status indicator replaces
the role badge. The sidebar logo also differs slightly. This suggests the
Edit Profile page may exist in a slightly different context (system-level
settings) or the branding was intentionally varied. Implement as shown.

The page is structured into three card sections:
1. Avatar & Identity — profile photo upload
2. General Information (left) + Security Vault (right) — name, email, password
3. Access Roles & Metadata — read-only system metadata

## User Access

- [ ] PUBLIC — not accessible
- [x] ADMIN — can edit any user's profile OR their own
- [ ] EDITOR — can only edit their own profile
  (EDITOR accessing another user's edit page → 403)

> Unauthenticated → redirect to 1.png.

## Permissions

**ADMIN can:**
- Edit any user's full name
- Edit any user's email address
- Edit any user's password (set a new password on behalf of user)
- Upload / replace any user's avatar photo
- View (read-only) Access Roles & Metadata section

**EDITOR can:**
- Edit only their own full name
- Edit only their own email address
- Change their own password (must enter current password)
- Upload their own avatar
- Cannot edit another user's profile

## Related Pages

- 13.md (User Details) → "Edit Profile" button navigates here
- 12.md (Users List) → pencil icon navigates here
  → "Cancel" returns to the referring page

### API Endpoints

```
GET /api/admin/users/:id
  → Load current values to pre-fill the form

PATCH /api/admin/users/:id
  Request: {
    full_name:        string (if changed)
    email:            string (if changed)
    current_password: string (required only when changing own password)
    new_password:     string (if changing password)
    photo_url:        string (if changed via upload)
  }
  → Partial update — only send changed fields
  Response: updated user object
  Errors:
    422 — email already in use
    422 — current password incorrect
    422 — new password too weak
    422 — new password same as current

POST /api/admin/upload/presigned-url
  Request: { filename, content_type, folder: "users/avatars" }
  Response: { presigned_url, public_url, key }
```

### Business Rules

- ADMIN can edit any user without entering the user's current password
- EDITOR can only edit their own profile — attempts to edit others → 403
- Email must be unique — backend checks on save
- Password change is optional — empty password fields = no password change
- Password minimum: 12 characters + complexity requirements
- When ADMIN sets a new password for another user, that user should be
  prompted to change it on next login (set `force_password_change = true`)
- Photo upload happens before form submission — stored in form state as URL
- All three sections save together on single "Update Profile" click
- Last Login, MFA Status, Access Tier are read-only — cannot be changed here
- Access tier is determined by system role — not manually editable

---

### Edge Cases

- User not found (deleted) → redirect to /admin/users with toast
- EDITOR accessing another user's edit page → 403 → redirect to dashboard
- Email conflict on save → inline error under email field:
  "This email is already in use by another account"
- Weak password → inline error: "Password must be at least 12 characters
  with uppercase, lowercase, number, and special character"
- Same as current password → inline error: "New password must differ from current"
- Wrong current password → inline error: "Current password is incorrect"
- Photo upload fails → toast error, form can still save (without new photo)
- No changes made, user clicks Update Profile → save still fires (idempotent)
