# Admin Blogs — Content Architecture (Blog Inventory)

## Business Description

The central blog management hub for the MFRA admin panel. Admins and editors
can view, filter, create, edit, and manage all blog posts from this page.
It presents the full blog inventory as a paginated table with status filtering,
category tags, ordering, and per-row actions.

The page also surfaces three real-time intelligence cards at the bottom —
engagement spikes, review requests, and content security monitoring — giving
editors contextual awareness without navigating away.

"Content Architecture" is the editorial branding for this section. The two-tone
heading ("Content" in white, "Architecture" in orange/primary) follows the
DESIGN.md editorial style.

## User Access

- [ ] PUBLIC — not accessible, redirect to login
- [x] ADMIN — full access: create, edit, delete, publish, reorder
- [x] EDITOR — can create, edit, publish own posts; cannot delete; cannot reorder

> Unauthenticated users redirected to 1.png (Login).
> EDITOR sees the same table but delete icon is hidden on each row.

## Permissions

View:
- Full blog inventory table (all statuses)
- Filter tabs: All Posts, Published, Drafts, Scheduled, Archived
- Impact metric: total published blogs count
- Insight cards at bottom
- Pagination

Create:
- "+ Create Blog" button → navigates to 5.png (Blog Editor, new mode)

Edit:
- Edit icon per row → navigates to 5.png (Blog Editor, edit mode)
- Order number field — ADMIN only: drag or inline edit to reorder

Delete:
- Delete icon per row — ADMIN only (hidden for EDITOR)
- Requires confirmation modal before deletion
- Soft delete — sets deleted_at, does not remove from DB

Other:
- Status filter tabs change the displayed list without page reload
- "Categories" button opens category management panel or modal
- "Last Modified" sort toggle — sorts table by updated_at descending
- "Sync AWS Cloud" button — future feature, non-functional in MVP (shows tooltip)
- Eye/view icon per row → navigates to 4.png (Blog Insights)
- Pagination — numbered, prev/next arrows

### API Endpoints

```
GET /api/admin/blogs
  Query params:
    status:   all | published | draft | scheduled | archived
    page:     number (default 1)
    size:     number (default 10)
    sort:     last_modified | created_at (default last_modified)
    order:    asc | desc (default desc)
  Response:
    data: array of blog summaries
      id, title, slug, cover_image_url, status, display_order,
      category_ids, categories[{id, name, slug}],
      created_at, updated_at, created_by_name, updated_by_name
    meta:
      total, page, size, total_pages, total_published

GET /api/admin/blogs/insights
  Response:
    top_blog_this_hour: { title, views }
    review_pending_count: number

PATCH /api/admin/blogs/{id}/order
  Body: { display_order: number }
  Auth: ADMIN only

DELETE /api/admin/blogs/{id}
  Auth: ADMIN only
  Soft delete — sets deleted_at
```

### Business Rules

- Table shows ALL statuses by default (All Posts tab)
- Filter tabs update URL query param — browser back button restores filter
- Default sort: last modified descending
- EDITOR role: delete icon hidden on all rows
- EDITOR role: ORDER column is read-only (no inline edit)
- Display order only changeable by ADMIN
- Delete requires confirmation modal: "Delete '[Blog Title]'? This cannot be undone."
  → soft delete, not hard delete
- After delete: row disappears from list, total count decrements
- "Sync AWS Cloud" button: visual only, no API call, tooltip "Coming soon"
- Category chips: if more than 2 categories, show first 2 + "+N more" pill
- Relative timestamps: "Updated 2h ago by Admin" — recalculate on load
- Thumbnail: if no cover image, show a placeholder with blog icon
- Blog title click and edit icon both navigate to 5.png in edit mode

---

### Edge Cases

- Empty state (no blogs match filter): show centered message "No blogs found"
  with a "+ Create Blog" CTA button
- If thumbnail image fails to load: show surface_container_highest placeholder
- If insight API fails: hide insight cards section silently — no error shown
- If blog count = 0 for "Published" tab: show empty state, not "0 Blogs Published"
  in the impact metric — still show the real total_published number
- Very long blog title: truncate to 2 lines with ellipsis
- Very many categories: show 2 chips + "+N" overflow pill
- Table loading: show skeleton rows (3 rows) while API call is in flight

---

### Out of Scope

- Bulk select and bulk delete
- Drag-and-drop reordering (order is editable inline by number only in MVP)
- CSV/PDF export of blog list
- Search within blog inventory (global search in top bar is placeholder only)
- Preview blog before publishing (done from 5.png editor, not from list)
- Scheduled publishing automation (Scheduled tab shows blogs but scheduling
  logic is managed from 5.png)
- Guest contribution workflow (Review Required card is informational only in MVP)
- DLP scan integration (static message in MVP)
- "Categories" full management (modal opens but category CRUD is a future feature)
